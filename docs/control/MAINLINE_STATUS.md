## Current Architecture North Star

StackChan is a bodily voice satellite / desktop Agent terminal.

Device side:

- Official StackChan firmware/body layer.
- Xiaozhi-compatible protocol.
- Microphone, speaker, Opus audio.
- Screen, expression, servos, LEDs, touch, camera.
- Device-side MCP tools.

Gateway side:

- Session state machine.
- Xiaozhi voice protocol ownership on the server side.
- ASR/LLM/TTS provider switching.
- Barge-in and generation cancellation.
- Memory, skills, local tools, search, Home Assistant.
- Permissions, rate limits, logs, latency metrics.
- Cloud deployment as the production path.

Agent/Skill side:

- Claude/Hermes/OpenClaw/V21 style systems connect as backend services.
- They do not own realtime audio or direct device output.

## Product Priority

1. P0: Reliable low-latency voice companion, half-duplex and interruptible first.
2. P1: Body toolization through MCP with clamp, rate limit, debounce.
3. P2: Desktop smart-home hub with Home Assistant integration.
4. P3: Personality and memory as gateway/service-side capabilities.
5. P4: Vision and embodied behavior via camera-as-tool calls.
6. P5: True realtime full-duplex after AEC/VAD/barge-in are proven.
