# Change Protocol

## 每次改动前

1. 阅读或定位相关计划任务。
2. 确认本次改动范围。
3. 确认是否需要 subagent。
4. 确认需要运行的验证命令。

## 每次改动中

- 小步修改。
- 不做无关重构。
- 不移动已稳定文档，除非任务要求。
- 不引入真实 provider 调用，除非当前任务就是 provider probe。
- 不把“以后也许需要”的抽象提前写进热路径。

## 每次改动后

1. 运行对应验证命令。
2. 更新 [TASK_BOARD.md](./TASK_BOARD.md)。
3. 汇总改动文件。
4. 说明未验证项。
5. 如用户要求提交，则一个任务一个 commit。

## 偏航处理

发现以下情况必须停下来重新对齐：

- 任务需要修改计划外的大面积文件。
- 测试失败超过两轮仍不清楚原因。
- 需要真实密钥但私有密钥文件不存在。
- 需要刷机、NVS 写入、物理设备操作。
- subagent 和主线程改动冲突。
- 新需求改变了 P0 边界。
